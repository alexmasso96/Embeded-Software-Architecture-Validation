Architecture Validator Pro — UI Design Survey

Six candidate designs for the new web-based UI. Every mockup is fully interactive with the same data and all six tabs, so you're comparing look, layout, and workflow — not features. All designs follow one shared rule: nothing hides behind right-click. Row actions live in a visible ⋯ menu (or an always-visible details panel), table actions live in a toolbar, and right-click will only ever be a shortcut.

Things to try in any design: switch tabs · click a Review/Port State pill to change it · open a row's ⋯ menu (try "Retire port…") · type in the search box to filter · press Add Port / Re-match / Save · edit the template in Test Design and watch the live preview (Previous/Next cycles ports) · press Generate in AI Generation to see SSE-style streaming · ask something in AI Chat · click nodes in the Code Map graph · click files in Change Log and run the AI summary.

Translucent sidebar · unified toolbar · light

2 · macOS Classic light

Big Sur/Monterey feel: source-list sidebar on the LEFT, segmented control for tabs, zebra-striped table, and a persistent action strip under the table showing the selected row's actions at all times.
Mica background · NavigationView · light

3 · Windows 11 Fluent light

Feels native on the corporate Windows laptops most of the team uses: left navigation rail with models embedded, a command bar attached to the table, page title with description.
Dense · keyboard-first · ⌘K command palette

4 · Linear Dense Dark dark

Engineering-tool aesthetic: compact rows, dot status indicators, filter chips, number-key tab switching, command palette, and a live activity feed in the right rail for collaboration visibility.
Activity bar · bottom details panel · status bar

5 · IDE Workbench dark

VS Code-style: icon activity bar, project explorer tree, editor tabs, and a bottom panel (Port Details / Match Candidates / History / Jobs) that makes the selected port's data and actions permanently visible.
White cards · breadcrumbs · review progress

6 · Clean Enterprise Light light

Atlassian-style light theme for audit-friendly readability: breadcrumbs, prominent page actions, filter chips, and a review-progress card — the most "management demo" friendly of the six.

Questions for the team

  1. Navigation: tabs along the top (1, 2, 6) or a navigation rail on the left (3, 4, 5)?
  2. Models list: own sidebar on the right (1, 6), merged into the left nav (3, 4), or a tree explorer (5)?
  3. Theme: dark (1, 4, 5), light (3, 6), or follow the OS with both?
  4. Row actions: ⋯ menu per row (1, 3, 4, 6), persistent action strip under the table (2), or a bottom details panel (5)?
  5. Density: comfortable rows (2, 3, 6) or compact engineering density (4, 5)?
  6. Status style: colored pills (1, 2, 3, 5, 6) or dot + text (4)?
  7. Which design would you trust in a customer/audit demo, and which would you prefer for 8 hours of daily work? (They may differ!)
